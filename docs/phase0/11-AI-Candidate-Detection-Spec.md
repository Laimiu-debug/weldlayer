# 11-AI-Candidate-Detection-Spec

## 1. 目标定位

当工程图中没有明确标注焊缝号、焊接符号或焊接说明时，系统不应直接产出“确定焊缝”，而应产出“候选焊缝”。

本规格定义导入解析阶段的 AI/规则联合识别方案，用于：

- 从 PDF / DWG 中识别疑似焊接连接位置
- 给出候选焊缝的证据、字段猜测和置信度
- 将候选结果送入人工确认流程

本规格不允许：

- 直接跳过人工确认进入正式匹配
- 用 AI 覆盖硬约束规则
- 在没有证据的情况下生成“确定焊缝”

核心原则：

`候选焊缝 != 正式焊缝`

## 2. 与现有系统主线的关系

当前主线为：

`项目 -> 图纸导入解析 -> 焊缝确认 -> 匹配 -> 导出`

在无焊缝标注场景下，导入解析页需要拆成两层结果：

1. 解析结果中的“显式焊缝”
- 图纸明确给出焊缝号、焊接符号、文字说明
- 可以直接生成待确认焊缝

2. 解析结果中的“候选焊缝”
- 图纸未明确标注，但存在较强几何/文字/上下文证据
- 只能进入候选复核队列

进入匹配页之前，必须经过人工确认，转成正式 `weld_seam` 记录。

## 3. 职责边界

### 3.1 本模块负责

- 构件连接点和疑似接头区域检测
- 焊接符号和说明文字 OCR / 结构化
- 邻近尺寸、材质、节点号、图号的证据关联
- 候选焊缝评分和排序
- 候选字段补全建议
- 证据留痕和人工复核支撑

### 3.2 本模块不负责

- 最终 PQR / 焊工资格 / 焊材匹配
- 代替工程师确认焊缝真值
- 自动放行到“可执行”结论

## 4. 输入范围

### 4.1 主输入

- PDF 图纸
- DWG 图纸

### 4.2 辅助输入

- 标题栏和图纸元数据
- 材料表 / BOM
- 技术说明和备注
- 节点详图 / 剖视图 / 局部放大图
- 同项目关联图纸

### 4.3 中间可用信息

- 几何实体：线、弧、多段线、块、图层
- 文本实体：尺寸、标注、说明、材料牌号
- OCR 文本片段
- 已识别焊接符号和引出线

## 5. 输出目标

系统输出的是 `weld_candidate[]`，不是最终 `weld_seam[]`。

每个候选项至少包含：

- `candidate_id`
- `draw_ref`
- `sheet_no`
- `page_index`
- `source_kind`
- `anchor_bbox`
- `candidate_type`
- `joint_geometry`
- `material_guess_a`
- `material_guess_b`
- `thickness_guess_a_mm`
- `thickness_guess_b_mm`
- `position_guess`
- `weld_symbol_guess`
- `confidence_score`
- `evidence[]`
- `review_status`

其中：

- `candidate_id` 用于导入页和确认页内部流转
- `weld_id` 只有在人工确认后才生成或绑定

## 6. 识别流程

### 6.1 几何候选生成

先不依赖 AI，先做规则候选生成：

- 相交线段检测
- 相贯管口检测
- 搭接边界检测
- 对接边缘检测
- 支管/接管/法兰邻接检测
- 同轴断开段连接检测

这一层的目标不是判断“是不是焊缝”，而是把可能的接头区域圈出来。

### 6.2 显式标注识别

对候选附近做证据抽取：

- 焊接符号
- 引出线箭头
- 文字说明
- 节点号
- 尺寸标注
- 材料和厚度说明

如果存在明确焊接符号和说明，则优先按显式焊缝处理。

### 6.3 上下文补全

对未显式标注的候选，补全可推断字段：

- 邻近构件材质
- 邻近厚度
- 构件类型
- 接头类型
- 可能位置编码
- 可能焊接方法提示

### 6.4 AI 评分与排序

AI 只做排序和补全，不直接作最终判定。

输入包括：

- 几何关系特征
- 邻近文本特征
- 图纸类型
- 候选局部截图或局部向量片段
- 关联详图上下文

输出包括：

- 是否疑似焊缝
- 候选接头类型
- 字段补全建议
- 置信度
- 解释理由

### 6.5 去重与合并

同一焊口可能在多个图层、多个视图、多个图纸中重复出现，需要做：

- 同图去重
- 跨视图合并
- 跨图关联
- 主次证据归并

最终保证一个候选焊缝只占一个复核槽位，但可挂多个证据来源。

## 7. 候选置信度分级

### 7.1 高置信度 `>= 0.85`

要求：

- 有明确几何接头证据
- 且存在邻近文字、尺寸或符号支撑中的至少一类

处理策略：

- 自动进入“待确认”列表
- 支持批量接受，但仍需要人工确认

### 7.2 中置信度 `0.60 - 0.85`

要求：

- 有较强几何证据
- 但文本/符号/字段补全不完整

处理策略：

- 默认标记为“可疑”
- 必须逐条确认

### 7.3 低置信度 `< 0.60`

要求：

- 几何关系不稳定或证据不足

处理策略：

- 默认不进入主确认表
- 仅在“低置信度候选”面板中展示
- 用户主动纳入后才进入复核流

## 8. 证据模型

每条候选必须带 `evidence[]`，至少包含：

- `type`
- `score`
- `source_ref`
- `summary`

建议证据类型：

- `geometry_intersection`
- `geometry_branch`
- `geometry_butt_joint`
- `geometry_lap_joint`
- `weld_symbol_detected`
- `leader_arrow_detected`
- `ocr_note_nearby`
- `material_label_nearby`
- `thickness_label_nearby`
- `detail_view_linked`

要求：

- 页面上能展开看证据
- 每次人工修改后要保留 before / after

## 9. 页面落地方式

### 9.1 第二页：图纸导入解析

新增候选视角：

- 文件级候选数
- 可疑候选数
- 低置信度候选数
- 候选筛选：显式 / 推断 / 可疑 / 已接受 / 已忽略

文件详情区需要支持：

- 点击候选热点高亮证据
- 查看候选字段猜测
- 查看证据列表
- 标记“是焊缝 / 不是焊缝 / 暂不确定”

### 9.2 第三页：焊缝信息确认

只接收两类数据：

- 已明确识别的焊缝
- 已人工接受的候选焊缝

第三页需要记录来源：

- `source_type = explicit | candidate`
- `candidate_id`
- `source_draw_ref`

这样后续匹配、审计、导出都能追溯到图纸来源。

## 10. 数据模型建议

建议新增独立实体 `weld_candidate`，不要直接把候选结果塞进 `weld_seam`。

最小字段：

- `candidate_id`
- `project_id`
- `draw_ref`
- `page_index`
- `source_kind`
- `candidate_type`
- `joint_geometry`
- `material_guess_a`
- `material_guess_b`
- `thickness_guess_a_mm`
- `thickness_guess_b_mm`
- `position_guess`
- `weld_symbol_guess`
- `confidence_score`
- `review_status`
- `accepted_weld_id`
- `evidence_json`
- `created_at`
- `updated_at`

推荐 `review_status`：

- `pending`
- `accepted`
- `rejected`
- `uncertain`
- `merged`

## 11. 工程约束

### 11.1 必须人工确认的场景

- 无焊接符号，仅靠几何推断
- 异种材料候选
- 厚度缺失或冲突
- 同一候选存在多个可能接头类型
- 多视图结果不一致

### 11.2 禁止自动放行的场景

- 候选结果没有证据列表
- 低置信度候选
- 字段补全来源不明确
- 无法定位回原图

## 12. 分阶段实现建议

### Phase A: 规则候选

先做：

- 几何候选点生成
- 简单 OCR 邻近匹配
- 候选热点展示
- 人工接受 / 拒绝

目标：

- 把“满图找焊口”变成“确认候选焊口”

### Phase B: AI 排序与补全

再做：

- 候选评分模型
- 接头类型判断
- 材质 / 厚度 / 符号补全建议
- 可读解释

目标：

- 减少人工逐条判断成本

### Phase C: 跨图合并

最后做：

- 总图 / 详图联动
- 候选归并
- 证据整合

目标：

- 提升复杂工程图场景下的稳定性

## 13. 验收标准

- 无焊缝标注图纸也能生成候选清单
- 每条候选都可回看原图位置和证据
- 高 / 中 / 低置信度分层清晰
- 未确认候选不能进入正式匹配
- 人工接受后的候选能进入第三页并保留来源追溯
- AI 故障时，规则候选流程仍能继续

## 14. 与 07-AI-Scope-v1 的关系

`07-AI-Scope-v1` 约束的是匹配阶段 AI：

- 推荐排序
- 冲突解释增强

本规格约束的是解析阶段 AI：

- 候选焊缝识别
- 候选字段补全
- 候选证据展示

两者共用同一条原则：

`规则优先，AI 辅助，人工最终确认`
