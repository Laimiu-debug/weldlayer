# 02-User-Flow

## 1. 主流程

1. 新建项目并选择标准体系（国标或 ASME）。
2. 导入工程图（PDF/DWG）并启动解析。
3. 生成焊缝信息表（含置信度）。
4. 焊接工程师人工确认并维护焊缝信息。
5. 系统基于标准规则匹配 PQR 与焊工资格。
6. 输出推荐结果、备选方案、冲突解释。
7. 选择工艺卡模板并生成预览。
8. 导出 Word/PDF 并归档日志。

## 2. 关键门禁

- 未确认焊缝信息不允许进入正式匹配。
- 硬约束冲突不允许直接生成“合规推荐”。
- 模板必填字段缺失时按策略阻断或警告。

## 3. 关键交互节点

- 图纸解析异常重试。
- 焊缝信息批量编辑与不确定项标记。
- 匹配失败后跳转 PQR/焊工资格维护页补数据。
- 冲突解释查看条款与修复建议。

## 4. 状态流转

- 焊缝条目：`pending -> confirmed/changed`
- 匹配结论：`match/partial/fail`
- 许可证：`NotActivated -> Activated -> GracePeriod/Expired`

## 5. 输出物

- 焊缝信息确认记录
- 匹配报告（推荐 + 冲突）
- 工艺卡（预览、Word、PDF）
- 审计日志（含 trace_id）
