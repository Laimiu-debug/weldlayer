# 01-PRD-v1

## 1. 产品目标

为焊接工程师提供一款本地化软件。用户输入工程图（PDF/DWG）后，系统自动提取焊缝信息，匹配企业已有 PQR 与焊工资格，输出可审核工艺卡，并提供可解释的冲突原因。

## 2. 目标用户

- 焊接工程师（主用户）
- 工艺审核人员
- 企业质量管理人员

## 3. 业务痛点

- 图纸信息提取耗时且易漏项。
- PQR 与焊工资格分散，人工匹配效率低。
- 不同标准体系下匹配规则复杂，解释难以追溯。
- 工艺卡模板因企业差异大，复用困难。

## 4. v1 范围（MVP）

- Windows 本地单机版。
- 输入支持 PDF、DWG。
- 支持复杂焊接符号与剖视信息识别。
- 生成焊缝信息表并支持人工确认维护。
- 内置 PQR 模块与焊工资格模块（可维护数据库）。
- 支持国标与 ASME（用户手动选择体系）。
- 智能匹配推荐 + 冲突解释（定位到字段/条款）。
- 生成可审核工艺卡，支持软件内预览、Word、PDF。
- 预设模板 + 企业自定义模板字段映射。
- 许可证支持在线激活 + 离线许可证文件激活。

## 5. v1.1 范围

- 局域网多人共享一库。
- 并发与基础权限控制。

## 6. 后续范围（非 MVP）

- 云化部署与 SaaS 化。
- 规则标准包商业化扩展。
- 组织级审批流和签章能力。

## 7. 关键约束

- 规则结论必须可审计、可追溯。
- AI 不能覆盖硬约束合规判定。
- 离线模式下核心流程可用。

## 8. 成功标准

- 能稳定完成“导入 -> 提取 -> 确认 -> 匹配 -> 导出”主流程。
- 冲突解释可定位到字段与条款。
- 工艺卡可被审核人员直接使用。
