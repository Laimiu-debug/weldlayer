# 15-Requirement-Decision-Log

## 1. 项目定位

- 面向焊接工程师的工艺匹配软件。
- 输入工程图后，自动匹配企业 PQR 与焊工资格。
- 输出可审核工艺卡。

## 2. 已确认决策

### 2.1 输入与解析

- 支持格式：PDF、DWG
- 解析能力：包含复杂焊接符号与剖视信息

### 2.2 数据管理

- 各企业源数据不统一
- 软件内必须提供 PQR 模块与焊工资格模块
- 字段需可配置（可选、可增删改）

### 2.3 标准体系

- v1：国标 + ASME
- 匹配方式：用户手动选择体系
- 规则模块：按热插拔设计，支持后续扩展与商业化

### 2.4 业务流程

- 先提取焊缝信息表
- 焊接工程师进行人工确认维护
- 再执行正式匹配

### 2.5 输出要求

- 可审核工艺卡
- 输出形态：软件内预览 + Word + PDF
- 模板能力：预设固定模板 + 企业字段映射模板

### 2.6 AI 能力

- v1 必要能力：智能匹配推荐、冲突解释
- 冲突解释粒度：精确到条款级
- 支持离线运行

### 2.7 部署与版本计划

- v1：Windows 本地单机
- v1.1：局域网多人共享一库
- 后续：SaaS 方向

### 2.8 许可证

- 必须支持在线激活
- 必须支持离线许可证文件激活

### 2.9 非需求项（v1 不做）

- 不做电子签章
- 不做审批流

## 3. 变更控制建议

- 任何范围变更先更新本文件。
- 变更通过后同步更新 `phase0/` 对应规范文档。
