# 14-UI-Navigation-Wireframe

## 1. 导航结构

1. 项目中心  
页面：项目首页、图纸导入、焊缝确认、匹配结果

2. 资源库  
页面：PQR 管理、焊工资格管理、焊材库与库存、模板映射

3. 输出中心  
页面：工艺卡预览与导出

4. 系统中心  
页面：许可证（系统日志/设置预留）

## 2. 全局布局

- 顶栏：项目信息、标准体系、许可证状态、全局搜索
- 左栏：分组导航
- 主区：页面主体
- 右栏：详情抽屉（条款、库存批次、历史、证据）
- 底栏：状态与 trace_id

## 3. 低保真线框说明

### 3.1 项目首页

- 左：最近项目列表
- 中：新建项目表单
- 右：项目统计卡

### 3.2 图纸导入页

- 左：文件队列
- 中：图纸预览画布
- 右：解析参数与日志

### 3.3 焊缝确认页

- 上：筛选工具条
- 中：可编辑大表格
- 右：详情与修改历史

### 3.4 资源库页

- PQR：主表 + 快速编辑抽屉
- 焊工：主表 + 到期预警
- 焊材：主表 + 库存/临期面板 + 批次明细
- 模板：模板列表 + 映射规则面板

### 3.5 匹配结果页

- 上：结论摘要（match/partial/fail）+ 库存可执行状态
- 中：推荐卡与备选卡
- 下：冲突表（含条款）+ 库存告警表
- 右：条款详情与库存批次证据抽屉

### 3.6 导出页

- 中：工艺卡预览
- 右：完整性检查 + 库存摘要
- 下：导出历史

### 3.7 许可证页

- 上：授权状态卡
- 中：在线激活区
- 下：离线激活区（licreq/lic）

## 4. 交互原则

- 主流程优先，减少跨页成本。
- 冲突必须可追溯到字段和条款。
- 库存预警必须可追溯到焊材批次。
- 关键状态可一眼识别（颜色 + 文案）。
