# 14-UI-Navigation-Wireframe

## 1. 导航结构

1. 项目中心  
页面：项目首页、图纸导入、焊缝确认、匹配结果

2. 资源库  
页面：PQR 管理、焊工资格管理、模板映射

3. 输出中心  
页面：工艺卡预览与导出

4. 系统中心  
页面：许可证（系统日志/设置预留）

## 2. 全局布局

- 顶栏：项目信息、标准体系、许可证状态、全局搜索
- 左栏：分组导航
- 主区：页面主体
- 右栏：详情抽屉（条款、历史、证据）
- 底栏：状态与 trace_id

## 3. 低保真线框说明

### 3.1 项目首页

- 左：最近项目列表
- 中：新建项目表单
- 右：项目统计卡

### 3.2 图纸导入页

- 左：文件队列
- 中：图纸预览画布
- 右：解析参数与日志

### 3.3 焊缝确认页

- 上：筛选工具条
- 中：可编辑大表格
- 右：详情与修改历史

### 3.4 匹配结果页

- 上：结论摘要（match/partial/fail）
- 中：推荐卡与备选卡
- 下：冲突表（含条款）
- 右：条款详情抽屉

### 3.5 模板与导出页

- 模板映射：左模板列表，中映射画布，右规则配置
- 导出预览：中预览，右完整性检查，下导出历史

### 3.6 许可证页

- 上：授权状态卡
- 中：在线激活区
- 下：离线激活区（licreq/lic）

## 4. 交互原则

- 主流程优先，减少跨页成本。
- 冲突必须可追溯到字段和条款。
- 所有关键状态都可一眼识别（颜色 + 文案）。
