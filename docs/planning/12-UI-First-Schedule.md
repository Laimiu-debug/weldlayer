# 12-UI-First-Schedule

## 1. 调整目标

将桌面端 UI 前置，采用“前端与后端双轨并行”，先用 Mock 数据跑通主流程界面，后续再联调真实能力。

## 2. 调整后排期

1. W1-W2（03/09-03/22）  
信息架构、设计系统、组件规范

2. W3-W4（03/23-04/05）  
关键页面高保真原型与可点击交互稿

3. W5-W6（04/06-04/19）  
桌面端前端壳开发，接 Mock 数据跑通

4. W7-W10（04/20-05/17）  
并行开发图纸解析、规则引擎、主数据模块

5. W11-W14（05/18-06/14）  
联调替换 Mock，闭环验证

6. W15（06/15-06/21）  
许可证模块接入 UI

7. W16-W18（06/22-07/12）  
集成测试、性能优化、发布准备

## 3. UI 前置收益

- 尽早暴露流程和交互问题
- 提前稳定字段口径与页面结构
- 降低后期联调返工成本

## 4. 风险与控制

- 风险：前端字段定义与后端接口偏差  
控制：冻结 Data Dictionary，接口契约先行

- 风险：Mock 与真实规则结果差异  
控制：W10 前完成一次全链路对齐评审
